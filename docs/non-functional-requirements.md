# 非功能要求

## 性能

- 200+ 服务差异对比结果页面首屏加载目标：2 秒以内，基于真实后端数据与真实运行态快照。
- 服务表格必须支持分页或虚拟滚动预留。
- 日志面板按增量加载，避免一次性加载超大日志。
- 后端列表接口必须支持分页。

## 可用性

- 所有任务状态必须可刷新。
- 发布、部署任务失败时必须能定位到具体步骤。
- 失败服务必须能看到失败原因和建议动作。
- Agent 离线时，项目环境任务不能直接开始执行。

## 可维护性

- 第三方系统集成必须通过 adapter 层，不允许业务代码直接调用具体 SDK。
- 前端页面不能硬编码业务数据，业务数据必须来自真实后端 API。
- 状态枚举集中定义，前后端保持一致。

## 可观测性

- 后端日志使用结构化日志。
- 任务执行日志包含 taskId、stepId、environmentId、agentId。
- API 返回 requestId。

## 本地开发

- docker-compose 一键启动依赖服务。
- README 必须说明启动方式。
- 本地开发也必须连接真实后端、数据库和已配置的集成资源；服务发版选择 Jenkins Pipeline 时必须触发真实 Jenkins 构建，不提供模拟构建开关。
