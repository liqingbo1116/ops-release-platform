# 安全要求

## 凭证管理

- 不允许在前端保存真实 Jenkins、Harbor、K8s、Nacos、MySQL、MinIO 凭证。
- 后端保存凭证时必须加密，MVP 可先使用环境变量中的 master key。
- API 返回环境配置时，只返回脱敏信息和连接状态。

## Agent 注册

- Agent 注册 token 必须由 Agent 管理页面生成，并展示平台地址、有效期和二进制启动配置示例。
- Agent 注册 token 必须有有效期，默认 10 分钟。
- token 只能使用一次。
- token 泄露后可废弃并重新生成。
- Agent 首次注册后使用独立 agent secret，不继续使用注册 token。
- Agent 心跳、任务租约、远程探测和执行回传接口必须校验独立 agent secret。
- 未绑定/待认领 Agent 即使在线，也不能拉取发布/部署任务。

## Agent 任务执行

- Agent 只能拉取已绑定项目/产品部署范围内的任务。
- 所有 shell 脚本步骤必须记录脚本名称、参数、执行人、执行时间、退出码和日志。
- 不允许平台直接拼接任意 shell 命令下发给 Agent。
- 脚本执行应来自平台托管脚本或预注册脚本模板。
- 高风险步骤必须支持人工确认。

## 操作审计

必须记录：

- 操作人
- 操作时间
- 环境
- 服务
- 变更前 tag
- 变更后 tag
- Agent
- 任务 ID
- 结果

## 发布安全

- 不可发布服务必须禁止勾选。
- workload 异常时默认跳过，不自动修复。
- 自动回滚只允许回滚到平台记录的上一 tag。
- 目标环境存在但来源基线不存在的服务，V1 只提示，不自动删除。

## 接口安全

- MVP 至少保留用户、角色、环境权限的模型。
- 后续接入统一登录或 SSO。
- 所有写操作需要记录审计日志。
