# MVP 验收标准

## 环境与 Agent

1. 能查看环境列表，包含环境类型、K8s/namespace、Harbor/project、Jenkins/view、Agent 状态和健康状态。
2. 能打开环境详情抽屉。本地环境显示平台基础资源关联和连接测试入口；远程环境显示 Agent 状态和最近上报时间。
3. 能查看 Agent 列表，包含版本、心跳、能力、最近任务。
4. 能打开 Agent 注册抽屉，看到一次性 token 和安装指令。

## 环境基线

5. 能查看环境基线列表。
6. 能进入基线详情查看服务镜像清单。
7. 能展示服务 namespace、workload、tag、digest、副本和健康状态。

## 差异对比

8. 能选择来源基线和目标环境生成差异对比。
9. 能筛选需更新、目标缺失、workload 异常、不可发布服务。
10. 文本搜索和状态筛选必须同时生效。
11. 不可发布服务不能被勾选。
12. 批量选择可发布服务时，必须跳过不可发布服务。

## 发布

13. 能创建发布单。
14. 能选择发布服务范围。
15. 能展示发布前风险提示。
16. 能提交发布并进入发布详情。
17. 能查看发布步骤、Agent 日志、失败定位建议。
18. 能对失败服务执行重试入口展示。

## 部署

19. 能查看部署任务列表。
20. 能进入部署任务详情。
21. 能展示部署步骤编排、当前步骤日志、人工确认节点。
22. 能展示重试、跳过、人工确认操作入口。

## 登录与权限

23. 未登录访问业务页面时，必须跳转登录页。
24. 能使用 mock 用户登录并进入首页工作台。
25. 顶部栏能展示当前用户名称、角色和退出登录入口。
26. 退出登录后清理本地 token，并回到登录页。
27. 能查看用户列表、角色列表和环境权限配置页面。
28. 基础写操作按钮能根据当前 mock 角色展示或禁用。

## 更新日志

29. 能查看更新日志页面。
30. 更新日志按小版本展示版本号、上线时间、更新类型、发布人和更新内容。
31. 能按版本号、更新类型、关键词筛选更新日志。
32. 更新日志数据来自 `mocks/` 目录或后端 mock API。

## 非功能

33. 前端工程可通过 `npm run dev` 启动。
34. 后端工程可通过 `go test ./...` 运行基础测试。
35. docker-compose 可启动 frontend、backend、postgres、redis，或按部署要求单独启动 postgres、redis。
36. V1 主线阶段完成的模块必须接真实后端数据并移除对应 mock/fallback；剩余未进入阶段的 mock 需要明确标记。
